# kotlin-swift-test

A minimal end-to-end proof that Kotlin can call Swift functions via JNI.

This demonstrates the output of the `--mode kotlin-jvm` generator: Swift functions are compiled into a native library, and Kotlin calls them through JNI bindings.

## Project Structure

```
kotlin-swift-test/
├── Sources/MyModule/
│   ├── MyModule.swift       # Swift functions
│   └── MyModuleJNI.swift    # JNI thunks (@_cdecl exports)
├── MyModule.kt              # Generated Kotlin bindings (external fun declarations)
└── Main.kt                  # Kotlin entry point
```

## Prerequisites

- Swift 6.2+
- Kotlin compiler
- JDK 21+

## How to Run

### Step 1: Compile Swift to a native library

```bash
swiftc Sources/MyModule/MyModule.swift Sources/MyModule/MyModuleJNI.swift \
  -module-name MyModule \
  -emit-library \
  -o libMyModule.dylib
```

### Step 2: Compile Kotlin

```bash
kotlinc MyModule.kt Main.kt -include-runtime -d app.jar
```

### Step 3: Run

```bash
java --enable-native-access=ALL-UNNAMED \
  -Djava.library.path=. \
  -jar app.jar
```

### Expected Output

```
add(2, 3) = 5
add(10, 20) = 30
isEven(4) = true
isEven(7) = false
DONE.
```

## How It Works

1. **Swift side** (`MyModule.swift`) — plain Swift functions with no annotations needed
2. **JNI thunks** (`MyModuleJNI.swift`) — `@_cdecl` functions that bridge JNI calls to Swift, manually written here but generated by `--mode kotlin-jvm` in the full tool
3. **Kotlin bindings** (`MyModule.kt`) — generated by `swift-java jextract --mode kotlin-jvm`, uses `external fun` declarations to call into the native library
4. **Main** (`Main.kt`) — calls the Swift functions as if they were regular Kotlin functions

## Relation to the Generator

The `MyModule.kt` file in this project is the direct output of running:

```bash
swift run swift-java jextract \
  --input-swift Sources/MyModule \
  --swift-module MyModule \
  --output-swift Sources/MyModule \
  --output-java . \
  --mode kotlin-jvm
```

The JNI thunks in `MyModuleJNI.swift` are generated by reusing the existing `JNISwift2JavaGenerator` thunk generation internally.
